Workshop 2 - Data

Data

Computer programs require data in order to function. In this workshop, we'll look at how to store data in our programs, and what different types of data there is.

Variables

A variable is a piece of memory that we allocate in which to store information. The memory system is fairly complex so we give our piece of memory a friendly name so we can access it easily. Have a look at this program. 


let myName = "Jimi";
let age = 30;

The keyword let tells the browser that I'm goin to create a variable and to free up some memory for me to use.

I then give my piece of memory a friendly name, myName and age are the variable names.

Then we use an = sign. This is an assignment operator and I am about to assign a value to my variable.

Lastly, I add my value and then end the operation with a semi-colon.

The variables are stored in memory for use later on in the program and we can retrieve the contents of those variables by using their names. For example, we could send them to the console log. 


let myName = "Jimi";
let age = 30;

console.log(myName);
console.log(age);

console.log (myName, age);

The console log will print the contents of the variable, not the variable name itself. You can also send multiple variables to the console log at once by separating them with commas.

Changing Variables

Variables are so named as they can change their values. Let's have a look at this program. 


let myName = "Jimi";
let age = 30;

console.log(myName);
console.log(age);

age = 40;
console.log(age);

In this program, the value of age has been updated. We did not need to use the let keyword again as the variable name had already been declared. variables can be updated.

Constant

For variables that should not change their values, we use a special variable called const and this will make the variable immutable. it can't be overwritten. 


let myName = "Jimi";
const age = 30;

console.log(myName);
console.log(age);

age = 40;

Try changing the age variable from let to const. What happens when you try to update a constant?

A TypeError occurs when you attempt to update a variable with information it can not hold. This error tells you that you cannot assign to a constant variable.

Using const allows JavaScript to understand that this piece of memory will not change. In small programs, there is not much difference, but in large systems, it can improve overall efficiency. It is good practice!

Var

You may see in some text books that var is used to declare variables. While this is acceptable to use, it is from an old version of JavaScript. Modern JavaScript should only use let or const.

Variable Names and Naming Conventions

For naming of variables, we follow the official documentation. This is to ensure good practice and standardisation.

Variable names can only contain numbers, letters, underscores and dollar signs. However, it must never start with a number.

Variable names should use camelCase - start with a lowercase letter and then use uppercase for each following word. Variable names can not contain spaces.


/* Examples of good variable names */
let car = "volkswagon";
let playerScore = 400;
let speed = distance / time;
let cashInDollars = 45

Lastly, it is good practice to name your variables so they are descriptive and can be identified easily in the context of the code. Your code forms part of your technical documentation and it should be understandable by other coders.

Data Types

In the previous examples, there were two variables, myName and age and myName held information within quote marks, but age didn't. This is because text and numbers are different data types. Let's take a look at some of the primitive data types that JavaScript uses.

Number

In JavaScript, the number datatype represents numeric values. This includes Integers(whole numbers) and Reals(Any number with a decimal point).


let integerValue = -5;  // Integers are whole numbers.
console.log(integerValue);

let realValue = 3.14; // Real numbers are numbers with decimal points. 
console.log(realValue);

Boolean

A boolean value only holds one bit of data, and that is either a 1 or a 0. For programming we know this as true or false. Booleans are mainly used for conditional statements. It can also be used as switches.


let lightSwitch = true;  // the light is on.
console.log(lightSwitch);

lightSwitch = false;     // the light is now off
console.log(lightSwitch);

String

A string data type is a row of text within quote marks. Strings are stored in memory as a sequence of characters. 


let fruit = "apple";
console.log(fruit);

let country = "Denmark";
console.log(country);

let phrase = "The quick brown fox jumped over the lazy dog.";
console.log(phrase);

Null

The Null datatype represents 'no value' or 'nothing.' It is different to 0 as 0 is a value (an integer). Null is the absense of any positive, negative, and zero value. Null is empty memory. 


let noNumber = null;
console.log(noNumber);

Undefined

The undefined data structure is reserved for unknown memory, or memory that has not been used - it does not even contain empty memory. 


let nothing; 
console.log(nothing);

In this example, the variable is reserved, but JavaScript has no idea if it is a number, a string, or a boolean - it is undefined.

Others

There are other data structures that we will look at later in the module. Symbol, Object, BigInt, Arrays are all complex data structures. Feel free to look up these data structures.

Arithmetic Operators

We've had a brief look at arithmetic before but it is important to see what this means in terms of variables and memory. Let's take a look at the common arithmetic operators. 


let num1 = 46;
let num2 = 32;

let sum = num1 + num2; // addition
let difference = num1 - num2; // subtraction
let product = num1 * num2; // multiplication
let quotient = num1 / num2; // division
let modulus = num1 % num2; // remainder.
let exponent = num1 ** num2; // to the power of.

num1++; // increment by 1.
num1--; // decrement by 1.
